# WordPress 提案用デモ実装 指示書（TablePress 最小構成）

## 目的

塾向けの学校情報ページ案件について、**クライアント提案用の最小デモ**を WordPress 上に実装してください。  
今回のデモで見せたいのは「完成版UI」ではなく、**来年以降も更新しやすい運用の骨格**です。

特に以下が伝わる状態を目指してください。

- 学校ごとの情報ページを作れる
- 毎年更新する表を **TablePress** で独立管理できる
- 投稿側から **関連する表ID** を紐付けられる
- 公開ページに表が正しく反映される

---

## 重要な前提

- これは **提案用デモ** です。完成版を作る必要はありません
- 実装は **最小構成** で構いません
- 依存プラグインは **TablePress のみ** を前提にしてください
- **ACF 前提にはしないでください**
- 追加プラグインなしで動くよう、**カスタム投稿タイプ + 独自メタボックス** で構成してください
- 既存テーマを大きく壊さず、**影響範囲を限定**してください
- CSS クラス名は衝突を避けるため、`school-demo__` プレフィックスを付けてください

---

## ゴール

以下が確認できるデモを実装してください。

1. 管理画面に「学校デモ」用のカスタム投稿タイプがある
2. 学校デモ投稿に、基本情報入力欄と **関連表ID入力欄** がある
3. 公開側で、学校情報 + TablePress の2表が表示される
4. TablePress が未インストールでも、画面が致命的に壊れず、プレースホルダー表示になる
5. コードが見通しよく整理されている

## 表の表示仕様

### 年度比較表
- メタボックスの `年度比較表ID` を使って表示
- TablePress が有効な場合は、該当テーブルを描画
- 無効な場合は「ここに年度比較表が表示されます」のようなプレースホルダーを表示

### 併願校表
- メタボックスの `併願校表ID` を使って表示
- 年度比較表と同様の方針で実装

### 実装方針
PHP 側では、TablePress 専用関数に強く依存しすぎず、可能ならショートコード経由で描画してください。  
例: `do_shortcode('[table id="12" /]')`

ただし実装しやすければ別の方法でも構いません。

## デザイン方針

### 方針
- 提案用デモなので、**見た目はシンプルで見やすく**してください
- 本格的なブランドデザインまでは不要です
- 表が主役なので、周辺UIは整理された落ち着いた見た目で十分です

### ざっくりした見た目の方向
- 横幅は読みやすい範囲に制限
- セクションごとに見出しを分ける
- 基本情報ブロックはカード風でもリスト風でも可
- 表の上下に適切な余白を入れる
- スマホでも崩れにくいようにする

### CSS命名
- 例: `school-demo__header`, `school-demo__meta`, `school-demo__section`, `school-demo__table`

## 実装上の注意

### 1. 既存テーマを壊さない
既存のテーマ構造がある前提で、なるべく独立した追加にしてください。  
既存テンプレートを大きく書き換えるより、専用テンプレートや専用ファイル追加を優先してください。

### 2. 依存を増やしすぎない
ACF や CMB2 前提にはせず、まずは標準の WordPress API で実装してください。

### 3. 入力値の保存
- 保存時は nonce チェックを行う
- autosave を考慮する
- 適切な sanitize を行う
- 出力時は escaping を行う

### 4. TablePress 未導入時の配慮
- Fatal にしない
- 管理画面も公開画面も壊れない
- 「TablePress を有効化するとここに表が表示されます」といった簡易文言で可

### 5. コメントを適度に残す
実装意図が分かる程度の簡潔なコメントを残してください。

## 推奨ファイル構成

既存テーマ配下で、必要に応じて以下のような構成にしてください。  
ファイル名は多少変えても構いませんが、役割分離は意識してください。

```text
theme/
├─ functions.php
├─ inc/
│  └─ school-demo.php
├─ templates/
│  └─ single-school_demo.php
└─ assets/
   └─ css/
      └─ school-demo.css
```

### 想定役割
- `inc/school-demo.php`
  - カスタム投稿タイプ登録
  - メタボックス追加
  - 保存処理
  - TablePress 編集リンク生成などの補助関数
- `templates/single-school_demo.php`
  - 公開側テンプレート
- `assets/css/school-demo.css`
  - 専用スタイル

既存テーマの都合上、テンプレート配置場所は調整して構いません。
